## Why I resist AI

When Copilot released autocomplete, I had a weird feeling develop inside me. It was helping me do
my job faster, but I resisted it. I was biased, so I only noticed tasks that it was bad at and used
that as justification to not use it at all. Then Agents was released inside VScode, followed by
Cursor and Claude. I wanted to resist even more, but the power of these tools was far too obvious now.
They were an objectively faster way to do a lot of my day to day stuff. Even then, every single time
there was an upgrade to the AI coding ecosystem, I felt weirdly sad. It was taking something away from me.

I used to love writing code by hand, and now that love is dying. There seems to be no point in
writing code without AI - it is a much faster way to get from an idea that I have in my head
to implementation. I am not talking about ideas for a new product or a new feature, but even just
the details of a small function. I can still turn off AI and write code myself - but I do not find
it satisfying to the same degree anymore. Something about the fact that I can get the same result
with much less effort is demotivating. The time has come where I have to force myself to think about
what I truly mean when I say "I like engineering" or "I like programming".

The thing that attracted me to programming was solving tricky problems myself - I valued being able
to describe the problem in code cleanly and figuring out bugs that others couldn't _all by myself_.
I took pride in the fact that I knew certain interesting details about my programming language that
my friends didn't. There was a piece of me in the program - it represented my ideas and beliefs on
computation, paradigms, languages and so on. And now there isn't, it is something purely mechanical.

One time during college, I had to write a program for an assignment in my C programming class. All my
experience with programming up until that point had been in the OOPS paradigm (Java) and I had no
no experience with C. It also so turned out that I hadn't started writing any code until the day before.
My day was spent going through C libraries and github projects to understand how to write code
the "C way" and then finally putting it down through Vim and then debugging. I did not sleep that night
and was so proud of myself when I finally got the program to work end to end. It was nothing too
special, but I understood every single line and had learnt so many new concepts and ideas in the
process.

Another fun thing, which might sound stupid to some, was that in the time I spent on C, I always LOVED
typing out `int main() { return 0; }`. That line signified a blank canvas, and the start of a journey
at the end of which I knew I would have learnt something new and interesting.

This is why I don't like AI. There was a world with something I valued in it, with which I had a deep
connect, and now that world is turning on its head. I used to think that the end product, the program
was the valuable part, but that was untrue. At least for me, I realize now that the process was far
more valuable. It was more or less the only thing I cared about.

You might ask why don't I just write programs the old way as a hobby - like a writer might still
write by hand, treating it as a craft. Truth is that I just don't find it satisfying anymore,
probably because while for me the value lies in the process, I also care about what others think of
me - as a programmer, as an engineer and as a human. This perception of others is affected by the end
product, irrespective of how I got there. AI is just a faster way to get to the same end product.
This results in a constant tension between my internal values about programming and my desire for
external validation. I wish it didn't.
